PlatformBrowser Trait (the only trait)

pub trait PlatformBrowser: Send + Sync {
    fn discover_browsers(
        &self
    ) -> Result<Vec<BrowserInfo>, DiscoveryError>;

    fn is_browser_running(
        &self, info: &BrowserInfo
    ) -> Result<bool, io::Error>;

    fn launch_url(
        &self, info: &BrowserInfo, url: &str
    ) -> Result<Child, LaunchError>;

    fn launch_new_tab(
        &self, info: &BrowserInfo, url: &str
    ) -> Result<Child, LaunchError>;
}

Launcher Launch Logic

fn launch_in_browser(&self, browser: &BrowserInfo, url: &str)
    -> Result<Child, LaunchError>
{
    match self.platform.is_browser_running(browser) {
        Ok(true)  => self.platform.launch_new_tab(browser, url),
        Ok(false) => self.platform.launch_url(browser, url),
        Err(_) => {
            // Detection failed — don't trust it, fall back.
            self.platform.launch_url(browser, url)
        }
    }
}

Platform Implementations

LinuxBrowser

Discovery: Checks $PATH for known executables via which crate. Falls back to common install locations. Queries xdg-settings get default-web-browser for the OS default.

Running detection: pgrep -x <exe_name>

Launch: Command::new(path).arg(url).spawn()

New tab: Same as launch — most Linux browsers open a new tab when given a URL.

MacBrowser

Discovery: Searches /Applications/*.app for known bundle IDs. Falls back to mdfind. Reads defaults for the default browser.

Running detection: pgrep -x <exe_name>

Launch: open -a <bundle_path> <url>

New tab: Same as launch — open -a opens a new tab if the app is already running.

WindowsBrowser

Discovery: Reads registry keys under SOFTWARE\Clients\StartMenuInternet. Checks HKCR\http\shell\open\command for the default.

Running detection: tasklist /FI "IMAGENAME eq <name>.exe"

Launch: Command::new(path).arg(url).spawn()

New tab: Same as launch — most Windows browsers open a new tab when given a URL.

Testing Strategy

MockPlatform — the test double

A pure-Rust implementation of PlatformBrowser that never touches the OS. It records whether browsers are "running" and which launch method was called. All unit tests for Launcher and BrowserRegistry use this.

Test Layer What It Tests How to Run
BrowserRegistry unit tests Lookup, preference setting, fallback to default cargo test launcher::registry::tests --lib
Launcher unit tests (with MockPlatform) Control flow: running → new tab, not running → launch, detection failure → fallback cargo test launcher::launcher::tests --lib
Platform unit tests Known executable/bundle ID lists, path validation, compilation check cargo test launcher::linux::tests --lib
Smoke tests (opt-in) Real browser discovery and process spawning on the current OS cargo test --features smoke-tests -- --ignored

Security Rules

No Shell Interpreters
Only std::process::Command::new(path).arg(url).spawn(). No sh -c, cmd /c, or powershell.
Path Validation
Before spawning, check executable_path.exists(). Return InvalidExecutable if the path is missing.
Pre-Validated URLs
The launcher assumes the caller has already validated the URL via url::parser. The launcher never parses or re-validates.
Zero Panics
No unwrap, expect, or panic! in production paths. Every failure returns a typed error.
